Novi HTML5 elementi stari browseri
HTML5 je uveo niz novih elemenata kao što su: header &gt;, footer &gt;, nav &gt;, article &gt;, section &gt; i time &gt;. Korištenjem novouvedenih elemenata web stranica postaje semantički jasnija. Na primjer, ako navigaciju web stranice stavimo unutar za to predviđenog nav &gt; elementa naš preglednik to može prepoznati i time olakšati snalažanje na web stranici slabovidnim i slijepim osobama. Osim toga semantički jasnija web stranica doprinosi optimizaciji sadržaja za tražilice (SEO). Međutim, ako ne koristimo suvremeni preglednik interneta koji ima podršku za HTML5, mogući su problemi s prikazom novih HTML5 elemenata.
Postavlja se pitanje kako će stari preglednici interpretirati nove, njima nepoznate HTML5 elemente? Preglednici se međusobno različito ponašaju po tom pitanju. Mozilla Firefox i Google Chrome će nepoznate elemente uredno prikazati i na njih će primjeniti CSS stilove koje sami definirate. Jedini nedostatak je što i Firefox i Chrome sve nepoznate elemente prikazuju inline, pa za HTML5 block elemente treba dodatno ručno definirati ispravan block prikaz:
Ako već koristite nekakav CSS reset stylesheet najbolje je dopuniti ga definicijama za HTML5. Već gotov CSS reset stylesheet možete naći ovdje.
Verzije Internet Explorer-a prije IE9 nepoznate elemente interpretiraju na vrlo problematičan način. Ako za nepoznate html elemente definirate CSS stilove IE ih neće uopće primjeniti. Osim toga IE ne dopušta da njemu nepoznati elementi sadržavaju druge elemente. Sadržaj nepoznatih elemenata će biti prikazan, ali će biti prikazan izvan nepoznatog elementa. Neće biti prikazan u očekivanom parent-child odnosu, nego će biti prikazani na istoj razini u hijerarhiji (siblings). Sam nepoznati element će biti prazan. Ovaj problem je ispravljen u IE9, ali postoji u starim verzijama čija upotreba je još uvijek vrlo raširena (IE8 i IE7). Tvorci starih verzija IE nisu predvidjeli pojavljivanje novih html elemenata, pa svaki html tag koji je IE nepoznat tretiraju kao grešku i neće ga prikazati.
Problem s prikazom HTML5 elemenata u IE je moguće riješiti javascript trikom. Na početku html dokumenta je potrebno javascript-om kreirati HTML5 elemente koji namjeravamo koristiti. Nije ih potrebno dodavati u dokument, dovoljno ih je kreirati. Javascript za kreiranje article &gt; elementa izgleda ovako:
script &gt; document.createElement (" article "); / script &gt;
Skriptu je najbolje staviti u zaglavlje html dokumenta, unutar head &gt; elementa. Ovo rješenje funkcionira za IE6, IE7 i IE8. Preporučuje se korištenje već gotovih skripti koje omogućuju korištenje svih HTML5 elemenata odjednom. Remy Sharp je autor jedne od najpopularnijih skripti. Možete je naći ovdje.
